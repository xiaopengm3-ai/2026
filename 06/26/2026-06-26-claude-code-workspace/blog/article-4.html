Windows 上的开发环境：2026 版

ToolingWindowsDevEnv

用了三年 macOS，去年切换回 Windows 11。有很多人不理解——"从 Unix 回到 Windows 不是自废武功吗？"——但 2026 年的 WSL2 已经足够好，好到我几乎感受不到差异，同时还能享受 Windows 的硬件兼容性和游戏体验。这篇文章整理了我的开发环境配置，希望对同样在 Windows 上做开发的朋友有帮助。

WSL2：类 Unix 体验的核心

WSL2 在 2026 年已经有了相当成熟的生态。它不是一个虚拟机模拟器，而是一个在 Hyper-V 轻量级虚拟机中运行的真实 Linux 内核。这意味着 Docker、systemd、GPU 直通都已经出奇地好用。

安装已经极其简单——一行命令：wsl --install。默认会安装 Ubuntu 24.04 LTS。我额外做了几个配置：

首先，在 WSL 的 /etc/wsl.conf 中禁止 Windows PATH 自动注入。这是很多性能问题的来源——Windows 的可执行文件路径被挂载到 WSL 中，导致每次 shell 启动时都要遍历 Windows 的 PATH，启动速度从 0.3 秒变成 3 秒：

[interop]
appendWindowsPath = false

其次，把代码放在 WSL 文件系统中而不是 /mnt/c。跨文件系统的 I/O 性能差异巨大——在 /mnt/c 上跑 npm install 比在 ~ 目录下慢了将近 10 倍。用 \\wsl$\ 路径可以在 Windows 端用 VS Code 直接打开 WSL 内的文件。

Windows Terminal + PowerShell 配置

Windows Terminal 是目前最好的终端模拟器之一，甚至比 macOS 的 Terminal.app 更好用。我的配置偏好：

终端配置——默认启动 WSL 的 Ubuntu profile，而不是 PowerShell。快捷键绑定了 Alt+D 分屏、Alt+Shift+D 垂直分屏。字体用 Cascadia Code——微软专门为终端设计的等宽字体，连字特性在阅读代码时非常舒服。

PowerShell Profile——虽然大部分时间在 WSL 里，但 PowerShell 仍然负责一些 Windows 特有的操作。我的 $PROFILE 配置了几样东西：

# 简洁提示符
function prompt { "$(Get-Location) > " }

# 常用 alias
function g { git $args }
function gst { git status }
function gco { git checkout $args }

# Starship prompt
Invoke-Expression (&starship init powershell)

用了 Starship 做跨 shell 的 prompt 主题——一套配置同时支持 PowerShell 和 WSL 里的 bash/zsh。Starship 在速度上比 oh-my-posh 快了一个数量级，因为它是 Rust 编译的原生二进制。

包管理器的选择

Windows 侧的包管理器我最终只留了两个：winget 和 scoop。两者定位不同：

winget 是微软官方的包管理器，适合安装"正经软件"——VS Code、Firefox、Docker Desktop 这类。它直接对接 Microsoft Store 和各个软件厂商的 CDN，可靠性最高。

scoop 是社区驱动的，适合安装开发工具——neovim、lazygit、ripgrep、fzf 之类的命令行工具。scoop 的特点是不需要管理员权限，所有软件装到用户目录下，干净无污染。更重要的是 scoop 支持 bucket 机制，extras bucket 里有很多社区维护的包：

scoop install neovim lazygit ripgrep fzf bat fd

Linux 侧（WSL 内）我用 mise 作为运行时版本管理器，替代了以前分散在各处的 nvm、pyenv、rbenv。mise 的一个配置文件 .mise.toml 就能管理 Node.js、Python、Go 的版本，比维护三套不同的版本管理器清爽很多。

工具推荐清单

VS Code——没有悬念的编辑器首选。WSL Remote 插件让你在 Windows 上写代码、在 Linux 上运行。Dev Containers 插件让你把整个开发环境声明为 Dockerfile，团队新成员加入时一键启动。

Claude Code——2026 年 AI 辅助编程的标配。从代码补全到 PR review、从 bug 修复到重构建议，深度整合到了日常 workflow 中。CLI 模式可以直接在终端中对话，不需要离开编辑器。

Everything——Windows 的杀手级文件搜索工具，NTFS MFT 索引级别的即时搜索。在 WSL 里找不到文件时切回 Windows 用 Everything 定位，一秒不到出结果。

PowerToys——微软官方的 Windows 增强工具集。最常用的三个功能：FancyZones 窗口管理（自定义分屏布局，比 Windows 11 原生的 Snap 灵活得多）、Keyboard Manager（重映射键位，我把 Caps Lock 映射为 Escape-Ctrl 双修饰键）、以及 Text Extractor（OCR 提取屏幕文字，类似 macOS 的 Live Text）。

总的来说，2026 年的 Windows 11 + WSL2 已经是一个出色的开发平台。Unix purist 可能会皱眉，但从工程实用主义的角度出发，它提供了目前最好的平衡——Linux 级别的开发体验，加上 Windows 的硬件生态和桌面软件兼容性。如果你在考虑切换，现在是个好时机。
