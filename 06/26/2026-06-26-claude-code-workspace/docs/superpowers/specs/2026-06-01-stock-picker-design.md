## 1. 项目概述与核心定位

### 1.1 项目名称

**A股智能选股系统（A-Share Intelligent Stock Screener）**

### 1.2 核心定位

一个面向个人投资者的**准机构级量化选股研究框架**。系统每日从 A 股全市场 4000+ 只股票中，通过基本面、技术面、资金面、消息面、行业面的多维复合评分，筛选出 **1–2 只最高质量的标的**，并输出完整的打分依据、投资逻辑和风险提示。

### 1.3 与 V1.0 的关键区别

| 维度 | V1.0 | V2.0 |
|------|------|------|
| 因子维度 | 基本面 + 技术面 | 基本面 + 技术面 + 资金面 + 行业面 + 消息面 + 催化剂 |
| 数据处理 | 简单过滤 | 去极值 + 标准化 + 行业中性化 + 市值中性化 |
| 防作弊 | 无 | 防未来函数 + 幸存者偏差处理 + 数据可用性校验 |
| 市场适配 | 无 | 牛熊识别 + 动态仓位建议 + 因子权重自适应 |
| 卖出逻辑 | 无 | 时间止盈 + 技术止损 + 基本面止损 + 分级减仓 |
| 因子评估 | 无 | IC/ICIR + 分层回测 + 因子贡献率 + 自动降权淘汰 |
| AI 能力 | 无 | 大模型资讯摘要 + 情绪分析 + 投资逻辑生成（多模型可插拔） |
| 风险控制 | 简单排雷 | 流动性过滤 + 行业集中度 + 黑名单 + 异常公告过滤 |
| 输出 | 打分卡 | 打分卡 + 投资逻辑 + 风险提示 + 推荐等级(S/A/B) + 周期建议 |

### 1.4 核心约束（不变）

- **市场**：仅 A 股（沪深两市）
- **交互方式**：命令行（CLI）
- **配置方式**：YAML 驱动，改权重/阈值不改代码
- **每日输出**：1–2 只精选标的，宁缺毋滥
- **回测**：内置历史回测验证打分有效性

### 3.2 数据质量保障机制

| 机制 | 说明 |
|------|------|
| **缺失值处理** | 财务数据缺失 → 标记而非填充，因子计算时跳过该股票在该因子上的得分 |
| **异常值检测** | 单因子值超出前后各5倍MAD范围 → 标记为异常，以行业均值替代 |
| **停牌处理** | 停牌股票在停牌期间不参与筛选，复牌后需观察3天重新纳入 |
| **复权检查** | 行情数据使用后复权价格，检测复权因子是否异常跳变 |
| **数据时效校验** | 每个数据源检查最后更新日期，超过阈值告警并降级到备用源 |
| **多源交叉验证** | 关键财务数据（营收、利润）与东方财富交叉比对，偏差>5%标记 |

### 3.3 本地缓存策略

```
data/cache/
├── market/           ← 行情缓存（Parquet，按代码+日期索引）
├── financial/        ← 财务缓存（Parquet，按代码+报告期索引）
├── valuation/        ← 估值缓存（Parquet，按日期索引）
├── fund_flow/        ← 资金流向缓存（Parquet）
├── industry/         ← 行业分类+估值缓存
├── news/             ← 资讯缓存（JSON，按日期索引）
└── meta.json         ← 缓存元数据：各数据集最后更新时间
```

| 缓存策略 | 说明 |
|---------|------|
| **行情数据** | 按日增量追加，每个代码一个 Parquet 文件。周五运行同时缓存当周周线 |
| **财务数据** | 按报告期全量覆盖写入，每个代码一个文件。检测到新财报发布时自动刷新 |
| **估值数据** | 每日全量覆盖（数据量小） |
| **行业分类** | 按季度刷新（申万行业分类每季度调整） |
| **资讯数据** | 按日追加，保留最近 30 天，定期清理 |
| **过期检测** | 读取缓存时检查 `meta.json` 中的最后更新时间，财务数据超过 1 天未更新标记告警 |

### 3.4 API 容错与降级策略

```
优先级：本地缓存 → akshare → 东方财富爬虫 → 跳过该数据

单接口超时 30 秒 → 重试 1 次 → 仍失败 → 降级
连续 3 次失败 → 该数据源标记为不可用（本次运行）
不可用数据超过 3 类 → 告警并退出（不输出不完整结果）
```

## 4. 量化严谨性体系

### 4.1 防未来函数机制

**这是整个系统最重要的数据治理原则。任何因子不得使用其计算时点之后的信息。**

| 机制 | 实现方案 | 说明 |
|------|---------|------|
| **财务数据公告日期约束** | 每个财务数据点记录其实际公告日期（非报告期截止日），T日选股仅使用公告日期 ≤ T 的数据 | 财报截止日可能是12月31日，但实际公告可能在次年4月。在3月选股时，不能使用4月才公告的年报数据 |
| **T日选股T+1成交** | 回测中，T日收盘后产生的信号，统一按T+1日开盘价（考虑滑点）模拟成交 | 消除"看到信号立刻买入"的回测作弊 |
| **业绩预告时间戳** | 业绩预告接口必须返回公告具体日期，系统只使用已公告的预告 | 预告虽在财报前发布，但也要锁定日期 |
| **分析师预期时间戳** | 盈利预测数据带预测发布日期 | 不使用"当前一致预期"概念，回测时只用历史的预测值 |
| **行业分类时间戳** | 记录历史上的行业分类变更日期，T日使用当时生效的分类 | 防止用现在的分类去选当时的股票 |
| **历史数据可用性校验** | 回测到某历史日期时，先检查哪些股票在该日期已有数据，只使用当时确实存在的数据 | 新上市的股票回测时不能出现在早期的选股池中 |

### 4.2 幸存者偏差处理

| 场景 | 处理方案 |
|------|---------|
| **退市股票** | 维护完整的退市股票列表及退市日期。回测期间包含已退市股票，退市当天按退市价格清算。退市前最后一个交易日正常参与选股 |
| **ST 历史状态** | 维护 ST 状态变更日志。回测到某历史日期时，使用当时的 ST 状态。正常股票→ST（某日）→退市，这条路径完整保留 |
| **历史行业分类** | 维护历史上的申万行业分类变更记录。T日使用T日生效的分类，不是当前分类 |
| **历史成分股** | 指数成分股（沪深300等）半年调整一次，回测时使用当时有效的成分股列表 |

### 4.3 行业中性化

**目的**：消除因行业属性导致的因子值系统性偏差。例如银行天然高杠杆、科技股天然高PE，不中性化会导致某些行业系统性被低估或高估。

| 步骤 | 方法 | 说明 |
|------|------|------|
| 1. 行业内排名 | 在每个申万一级行业内，对因子值排序，计算行业内百分位排名 | 将绝对因子值转化为相对排名 |
| 2. 行业均值偏离 | 计算每只股票的因子值相对于其行业均值的偏离度（ZScore形式） | 衡量该股票在行业内处于什么位置 |
| 3. 行业暴露控制 | 最终组合中，单一行业的权重不超过 30%（可配置） | 防止过度集中在某个行业 |

**适用因子**：PE、PB、ROE、资产负债率、毛利率等受行业属性强影响的因子。
**不适用因子**：动量类、量价类技术因子（这些通常跨行业可比）。

### 4.4 市值中性化

**目的**：消除市值规模对因子表现的干扰。大盘股和小盘股在很多因子上呈现系统性差异。

| 步骤 | 方法 |
|------|------|
| 1. 市值分组 | 将全市场股票按流通市值分为 5 组（超大盘/大盘/中盘/小盘/微盘），在组内进行因子排名 |
| 2. 市值回归残差 | 对受市值影响显著的因子，做横截面回归：`factor = α + β × ln(market_cap) + ε`，取残差 ε 作为市值中性化后的因子值 |
| 3. 双中性化 | 对关键核心因子，同时做行业中性化 + 市值中性化 |

### 4.5 因子标准化

| 步骤 | 方法 | 说明 |
|------|------|------|
| **去极值（MAD法）** | 计算因子值的中位数 median 和 MAD（median absolute deviation），将超过 `median ± 5×MAD` 的值压缩到边界值 | 防止极端值拉偏整个分布 |
| **标准化（ZScore）** | 去极值后做 ZScore 标准化：`(x - μ) / σ` | 使不同量纲的因子可比 |
| **缺失值处理** | 财务数据缺失 → 该因子不参与该股票的评分。技术数据缺失（上市不满周期）→ 该因子得分为行业中位数 | 区分"数据差"和"没数据" |

**ZScore 适用性**：
- 去极值后做 ZScore → 适用（分布基本对称的因子）
- 去极值后做 Min-Max → 适用（有明确物理意义的因子，如换手率）
- 直接百分位排名 → 适用（分布高度偏态的因子，如 PE、商誉/净资产）

### 4.6 数据版本管理

| 维度 | 方案 |
|------|------|

## 6. 基本面因子模块

### 6.1 估值因子（权重 35%）

| 子因子 | 子权重 | 满分条件 | 零分条件 | 中性化 | 评分方式 |
|--------|--------|---------|---------|--------|---------|
| PE 行业分位（5年） | 30% | 分位 ≤ 10% | 分位 ≥ 80% 或 PE≤0 | 行业内 | 百分位映射 |
| PB 行业分位（5年） | 25% | 分位 ≤ 10% | 分位 ≥ 80% 或 PB≤0 | 行业内 | 百分位映射 |
| PS 行业分位（5年） | 20% | 分位 ≤ 10% | 分位 ≥ 80% | 行业内 | 百分位映射 |
| PE 绝对值 | 15% | PE ∈ [5, 15] | PE > 50 或 PE < 0 | 行业+市值 | ZScore |
| 股息率 | 10% | ≥ 3% | 0% | 行业 | 线性映射 |

**设计说明**：
- PE/PB/PS 全部使用**同行业 5 年历史分位数**作为基准，不做跨行业绝对值对比
- 分位数计算基于该行业所有成分股过去 5 年的日频数据
- 对于近 5 年内上市的股票，使用上市以来的数据计算分位数，并在评分上给一个小的惩罚项（-5分），反映历史数据不足的不确定性
- PE/PB≤0 的股票不参与估值评分，需依赖其他维度突围

### 6.2 成长因子（权重 25%）

| 子因子 | 子权重 | 满分条件 | 零分条件 | 评分方式 |
|--------|--------|---------|---------|---------|
| ROE (TTM) | 30% | ≥ 20% | ≤ 5% | 行业内 ZScore |
| ROE 趋势（连续3季环比方向） | 15% | 连续3季上升 | 连续3季下降 | 离散打分(0/33/67/100) |
| 营收同比增速 | 20% | ≥ 20% | ≤ 0% | 行业内+市值中性 ZScore |
| 净利润同比增速 | 20% | ≥ 25% | ≤ 0% | 行业内+市值中性 ZScore |
| 毛利率趋势（连续2季环比） | 15% | 连续2季提升 | 连续2季下降 | 离散打分(0/50/100) |

**ROE 趋势详解**：
- 取最近 4 个季度的 ROE（单季）值
- 计算相邻季度变化方向（↑ 或 ↓）
- 3 次比较中：3↑ = 100分、2↑1↓ = 67分、1↑2↓ = 33分、3↓ = 0分

### 6.3 质量因子（权重 25%）

| 子因子 | 子权重 | 满分条件 | 零分/淘汰条件 | 评分方式 |
|--------|--------|---------|-------------|---------|
| 资产负债率 | 25% | ≤ 40% | ≥ 70%（高杠杆硬淘汰） | 行业内 ZScore |
| 经营现金流/净利润 | 30% | ≥ 1.0 | ≤ 0.3 | 行业内 ZScore |
| 商誉/净资产 | 25% | ≤ 10% | ≥ 50%（硬淘汰） | 行业内百分位 |
| 股权质押比例 | 10% | ≤ 20% | ≥ 60%（硬淘汰） | 线性映射 |
| 近3月大股东减持 | 10% | 无减持 | 近1月有大额减持(>5000万) | 离散打分 |

**硬淘汰规则**（先于打分执行）：
- 资产负债率 ≥ 70% → 淘汰（金融行业例外：不适用此条）
- 商誉/净资产 ≥ 50% → 淘汰
- 股权质押 ≥ 60% → 淘汰
- 净利润连续 2 季度为负 → 淘汰
- ST / *ST → 淘汰
- 近 1 个月收到证监会立案调查 → 淘汰

**金融行业特殊处理**：
- 银行/保险/券商不适用"资产负债率"指标
- 替换为：不良贷款率（银行）、偿付能力充足率（保险）、净资本/净资产（券商）

### 6.4 预期因子（权重 15%）

| 子因子 | 子权重 | 满分条件 | 零分条件 | 数据时效约束 |
|--------|--------|---------|---------|------------|
| 业绩预告方向 | 30% | 预增/扭亏/续盈 | 预减/首亏/续亏 | 公告日期 ≤ T日 |
| 分析师预期上调（近1月） | 25% | 上调家数 > 下调家数 × 2 | 下调家数 > 上调家数 | 预测发布日期 ≤ T日 |
| 预收账款/合同负债环比变化 | 20% | 环比↑ > 20% | 环比↓ > 10% | 最近可得的财报数据 |
| 经营现金流趋势 | 15% | 连续2季改善 | 连续2季恶化 | 最近可得的财报数据 |
| 管理层增持 | 10% | 增持金额 > 500万 | 无增持或有减持 | 公告日期 ≤ T日 |

### 6.5 基本面总分公式

```
估值得分 = Σ(估值子因子得分 × 子权重)    [0-100]
成长得分 = Σ(成长子因子得分 × 子权重)    [0-100]
质量得分 = Σ(质量子因子得分 × 子权重)    [0-100]
预期得分 = Σ(预期子因子得分 × 子权重)    [0-100]

基本面总分 = 估值得分 × 0.35 + 成长得分 × 0.25 + 质量得分 × 0.25 + 预期得分 × 0.15
```

| RSI 底背离 | 10% | 价格创新低而 RSI 未创新低 | 价格创新高而 RSI 未创新高（顶背离） |

### 7.4 形态因子（权重 10%）

**识别方式**：基于局部极值点 + 趋势线拟合的规则引擎，不使用深度学习。

| 看涨形态（加分） | 识别规则 |
|-----------------|---------|
| W 底 / 双底 | 两个相近低点（差距 < 5%），间隔 ≥ 20 个交易日，中间高点为颈线，颈线突破确认 |
| 头肩底 | 三底结构：左肩+头部(最低)+右肩，右肩不低于左肩，突破颈线上方 3% 确认 |
| 杯柄形态 | 上升后回调形成 U 形杯体（回撤 < 35%），杯底盘整后形成下倾短柄（回撤 < 15%），柄部突破确认 |
| 上升三角形 | 水平上轨 + 逐步抬高的下轨，≥ 3 次触上轨 + ≥ 2 次触下轨 |
| 旗形突破 | 急涨（旗杆）+ 平行下行通道（旗帜）+ 放量突破上轨 |
| 底部盘整突破 | ≥ 30 天窄幅震荡（振幅 < 15%）+ 放量（量比 ≥ 1.5）突破盘整区间上沿 |

| 看跌形态（减分） | 识别规则 |
|-----------------|---------|
| M 头 / 双顶 | 两个相近高点，下破颈线 |
| 头肩顶 | 三峰结构：左肩+头部(最高)+右肩 |
| 下降三角形 | 逐步降低的高点 + 水平支撑 |

**打分规则**：
- 检测到 ≥ 2 个看涨形态且无看跌形态 → 100 分
- 检测到 1 个看涨形态且无看跌形态 → 70 分
- 无形态或涨跌形态并存 → 50 分
- 检测到看跌形态 → 30 分
- 检测到 ≥ 2 个看跌形态 → 0 分

### 7.5 多周期因子（权重 10%）

| 周期 | 权重 | 判断内容 | 看多条件 |
|------|------|---------|---------|
| 周线 | 50% | 定大方向 | EMA20(周) > EMA50(周)，ADX(周) ≥ 20，价格在 EMA200(周) 上方 |
| 日线 | 35% | 主力战场 | 各因子主要计算周期，综合日线趋势+动量得分 |
| 60分钟 | 15% | 精找买点 | RSI(60m) 不超买，MACD(60m) 金叉或底背离，不创新低 |

**多周期一致性打分**：

| 状态 | 得分 |
|------|------|
| 三周期一致看多（共振） | 100 |
| 周 + 日看多，60m 中性 | 70 |
| 仅日线看多 | 40 |
| 周线看空 | **技术面总分直接归零**（大周期铁律，不可逾越） |
| 三周期一致看空 | 0 |

### 7.6 多因子共振加成

共振加成在技术面总分上叠加，最高额外 +20%：

| 条件 | 加成 |
|------|------|
| 趋势 ≥ 60 且动量 ≥ 60 | +10% |
| 趋势 ≥ 60 且量价 ≥ 60 | +10% |
| 趋势 ≥ 60 且动量 ≥ 60 且量价 ≥ 60（三者共振） | +20%（取代前两条，不叠加） |
| 形态 ≥ 70 且多周期 ≥ 70 | +15% |

### 7.7 技术面总分公式

```
趋势得分 = Σ(趋势子因子得分 × 子权重)     [0-100]
动量得分 = Σ(动量子因子得分 × 子权重)     [0-100]
量价得分 = Σ(量价子因子得分 × 子权重)     [0-100]
形态得分 = 离散规则得分                    [0-100]
多周期得分 = 离散规则得分                   [0-100] 或直接归零

基础分 = 趋势×0.35 + 动量×0.25 + 量价×0.20 + 形态×0.10 + 多周期×0.10

技术面总分 = min(100, 基础分 × (1 + 共振加成比例))

IF 周线看空 THEN 技术面总分 = 0
```

## 8. 资金行为分析模块

### 8.1 模块定位

资金行为分析作为一个**独立打分维度**，权重占最终评分的 10%。该模块追踪市场资金的真实行为痕迹——"钱在往哪里去"。

### 8.2 北向资金分析

| 子因子 | 子权重 | 满分条件 | 零分条件 |
|--------|--------|---------|---------|
| 北向资金近 5 日净流入（占流通市值比） | 35% | 净流入/流通市值 ≥ 0.5% | 净流出 |
| 北向资金近 20 日累计净流入趋势 | 30% | 20 日持续净流入且流入加速 | 持续净流出 |

**数据来源**：akshare 沪深港通持股接口，每日更新。

**评分方式**：各子因子先做行业内 ZScore 中性化，再映射到 0-100 分。

### 8.3 主力资金分析

| 子因子 | 子权重 | 满分条件 | 零分条件 |
|--------|--------|---------|---------|

| 超大单净流入方向 | 25% | 近 5 日超大单持续净流入 | 超大单持续净流出 |
| 大单净流入方向 | 20% | 大单持续净流入 | 大单持续净流出 |
| 中单/小单流向验证 | 10% | 中单+小单净流出（散户在卖） | 中单+小单净流入（散户在接盘） |
| 资金流向与价格的关系 | 10% | 价平量增（资金悄悄进场） | 价涨量缩（上涨乏力） |

**资金分类**：
- 超大单：≥ 100 万元
- 大单：20 万 – 100 万元
- 中单：4 万 – 20 万元
- 小单：< 4 万元

**数据来源**：akshare 资金流向接口 + 东方财富资金流向爬虫，每日更新。

### 8.4 筹码结构分析

| 子因子 | 子权重 | 满分条件 | 零分条件 |
|--------|--------|---------|---------|
| 股东户数变化（季度环比） | 40% | 户数↓ > 10%（筹码集中） | 户数↑ > 20%（筹码分散） |
| 机构持仓变化（季度环比） | 40% | 机构持仓比例↑ > 2pp | 机构持仓比例↓ > 3pp |
| 户均持股市值 | 20% | 户均持股市值在同市值组前 30% | 后 30% |

**筹码集中解读**：
- 股东户数减少 + 户均持股增加 = 筹码从散户向机构/大户集中（正面信号）
- 股东户数增加 + 户均持股减少 = 筹码从机构/大户向散户分散（负面信号）
- 注意：需要区分"大股东减持导致户数增加"和"机构增持导致户数减少"

### 8.5 资金行为总分公式

```
北向资金得分 = Σ(北向子因子得分 × 子权重)    [0-100]
主力资金得分 = Σ(主力子因子得分 × 子权重)    [0-100]
筹码结构得分 = Σ(筹码子因子得分 × 子权重)    [0-100]

资金行为总分 = 北向得分 × 0.40 + 主力得分 × 0.35 + 筹码得分 × 0.25
```

## 9. 行业景气度模块

### 9.1 模块定位

行业景气度作为**独立评分维度**，权重占最终评分的 10%。该模块回答"这只股票所在的行业现在好不好"这个问题。

### 9.2 行业景气度因子

| 子因子 | 子权重 | 计算方式 | 说明 |
|--------|--------|---------|------|
| 行业指数相对强度 | 35% | 行业指数近 20 日涨幅排名（vs 其他申万一级行业） | 衡量行业近期表现 |
| 行业资金流向 | 25% | 行业内全部成分股的近 5 日主力资金净流入总和 / 行业总市值 | 钱是否流入这个行业 |
| 行业估值变化 | 20% | 行业 PE 中位数的近 60 日变化方向 | 正变化=行业估值扩张，负变化=收缩 |
| 行业热度/关注度 | 20% | 行业内成分股近 5 日平均换手率 / 全市场平均换手率 | 市场对这个行业的关注程度 |

### 9.3 行业评分卡

| 行业排名 | 得分 | 说明 |
|---------|------|------|
| 前 10% 行业 | 100 | 领先行业 |
| 前 10%-30% | 75 | 强势行业 |
| 前 30%-50% | 50 | 中性行业 |
| 后 50%-70% | 25 | 弱势行业 |
| 后 30% | 0 | 落后行业 |

### 9.4 行业景气度总分

```
行业得分 = Σ(子因子得分 × 子权重)    [0-100]

该股票继承其所属申万一级行业的得分。
如果股票跨行业经营（如集团公司），取主业所在行业的得分。
```

## 10. 资讯分析引擎

### 10.1 模块定位

资讯分析引擎建立**独立的消息面评分体系（News Score）**，权重占最终评分的 10%。该模块实现从新闻文本到量化分数的转化。

### 10.2 数据源体系

| 数据源 | 类型 | 内容 | 获取方式 | 权重 |
|--------|------|------|---------|------|
| 巨潮资讯 | 官方公告 | 定期报告、临时公告、问询函回复 | 爬虫 | 35% |
| 财联社 | 财经快讯 | 即时新闻、政策解读、行业动态 | 爬虫 | 25% |
| 中国证券报 / 证券时报 | 官方媒体 | 政策信号、市场评论、公司报道 | 爬虫 | 15% |
| 雪球 | 投资者社区 | 用户讨论、热门帖子、关注度 | 爬虫 | 15% |
| 东方财富股吧 | 散户社区 | 散户情绪、讨论热度 | 爬虫 | 10% |

### 10.3 新闻处理流水线

```
原始资讯文本
    │
    ▼
[1. 去重] → 同一条新闻可能在多个平台出现，按标题相似度 > 0.8 去重
    │
    ▼
[2. 实体识别] → 提取股票代码/名称，将新闻关联到具体股票
    │
    ▼
[3. 时间戳校验] → 确保资讯发布时间 ≤ 选股日期（防未来函数）
    │
    ▼
[4. LLM 分析] → 调用大模型进行情绪分析 + 利好利空分类 + 摘要生成
    │          （详见第 12 章）
    ▼
[5. 量化打分] → 将 LLM 分析结果转化为结构化分数
```

### 10.4 资讯评分体系

| 评分维度 | 子权重 | 评分规则 | 数据来源 |
|---------|--------|---------|---------|
| 近期利好新闻密度 | 35% | 近 7 日利好新闻数量 × 质量系数。质量系数：官方公告=1.0、正规财经媒体=0.8、社区=0.4 | LLM 分类结果 |
| 新闻情绪得分 | 30% | 近 7 日新闻中，利好/(利好+利空) × 100。考虑时间衰减（越近权重越高） | LLM 情绪分析 |
| 市场关注度 | 20% | 近 7 日相关新闻总数 / 同行业平均新闻数 | 去重后的新闻数量 |
| 社区情绪得分 | 15% | 雪球+股吧帖子情绪的日频均值。正面帖/(正面+负面) × 100 | LLM 社区情绪分析 |

### 10.5 消息面总分

```
消息面总分 = Σ(评分维度得分 × 子权重)     [0-100]

时间衰减函数: weight = e^(-0.1 × days_ago)
即: 今天的新闻权重=1.0, 3天前=0.74, 7天前=0.50

每日资讯数量不足（该股票近7日新闻<3条）→ 消息面得分不参与该股票评分
（即该维度的权重重新分配给其他维度）
```

## 11. 催化剂系统

### 11.1 模块定位

催化剂（Catalyst）是独立于常规信息的**事件驱动因子**，权重占最终评分的 5%。催化剂检测"可能改变市场预期的关键事件"，这些事件往往比常规新闻对股价的影响更大、更持久。

### 11.2 催化剂分类与评分

| 催化剂类型 | 得分权重 | 典型事件 | 影响持续性 | 识别方式 |
|-----------|---------|---------|-----------|---------|
| 业绩预增 | 25% | 业绩预告大幅预增（≥ 50%）；业绩快报超预期 | 中短期（1-3月） | 业绩预告接口 + LLM 分析 |
| 回购 | 15% | 公司发布股票回购方案，回购金额占市值 > 0.5% | 短期（1-2月） | 公告爬虫 |
| 增持 | 15% | 大股东/高管增持 > 500 万元；员工持股计划 | 短期（1-2月） | 公告爬虫 + 减持/增持接口 |
| 并购重组 | 15% | 重大资产重组、并购、借壳上市 | 中期（3-6月） | 公告爬虫 |
| 重大订单/合同 | 15% | 签署重大合同，金额占营收 > 30% | 中期（3-6月） | 公告爬虫 |
| 政策利好 | 15% | 国家/行业政策直接利好该股票所在行业 | 长期（6-12月） | LLM 政策分析 |

### 11.3 催化剂检测机制

| 检测步骤 | 说明 |
|---------|------|
| 每日公告扫描 | 扫描当日所有 A 股上市公司公告，匹配上述六类关键词 |
| LLM 验证 | 疑似催化剂的公告，由 LLM 确认是否为实质催化剂（过滤虚假/模糊利好） |
| 催化剂有效期 | 每个催化剂有有效期限：预增→3个月、回购→2个月、增持→2个月、并购→6个月、重大订单→3个月、政策→12个月。超过有效期自动过期 |
| 催化剂叠加 | 同一股票同时有多个催化剂 → 得分叠加，上限 100 分 |

### 11.4 催化剂得分计算

```
催化剂基础得分 = Σ(该股票有效催化剂得分 × 催化剂类型权重)

催化剂叠加奖励:
  1个催化剂 → 基础得分
  2个催化剂 → 基础得分 × 1.2（+20% 共振奖励）
  3个及以上催化剂 → 基础得分 × 1.5（+50% 强共振奖励）
  上限 100 分

无催化剂 → 催化剂得分 = 50（中性分，不给0分避免拉低总分）
```

## 12. 大模型 AI 分析层

### 12.1 模块定位

大模型 AI 分析层是一个**可插拔的服务层**，为系统提供三项核心能力：
1. 新闻/公告的情感分析和摘要
2. 催化剂事件的自动识别和验证
3. 最终选股结果的投资逻辑生成

该层通过统一接口支持多种模型后端，用户可在 `config.yaml` 中切换。

### 12.2 多模型支持架构

```
                  ┌──────────────────────┐
                  │    AIAnalysisClient  │
                  │    (统一接口层)       │
                  │                      │
                  │  analyze_news()      │
                  │  analyze_sentiment() │
                  │  identify_catalyst() │

                         │

         ▼               ▼               ▼               ▼
   ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
   │ OpenAI   │  │ Claude   │  │ Gemini   │  │ DeepSeek │
   │ Adapter  │  │ Adapter  │  │ Adapter  │  │ Adapter  │
   └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘
        │              │              │              │
```

### 12.3 统一接口定义

```yaml
# config.yaml 中 AI 模块配置
ai:
  provider: "openai"              # openai | claude | gemini | deepseek | local
  api_key: "${AI_API_KEY}"        # 环境变量引用
  model: "gpt-4o"                 # 模型名称
  
  # 降级策略
  fallback:
    enabled: true
    providers: ["deepseek", "local"]   # 主模型失败时的降级顺序
  
  # 功能开关
  features:
    news_sentiment: true          # 新闻情绪分析
    catalyst_detection: true      # 催化剂识别
    thesis_generation: true       # 投资逻辑生成
    risk_assessment: true         # 风险提示生成
    
  # 速率限制
  rate_limit:
    max_requests_per_minute: 60
    max_tokens_per_day: 1000000
    
  # 缓存
  cache:
    enabled: true
    ttl_hours: 24                 # 同一新闻24小时内不重复分析
```

### 12.4 各功能 Prompt 设计原则

| 功能 | 输入 | 输出 | Prompt 设计要点 |
|------|------|------|----------------|
| **新闻情绪分析** | 新闻标题+正文 | `{sentiment: "positive/neutral/negative", score: 0-100, category: "利好/利空/中性", summary: "50字摘要", key_points: [...]}` | 要求从投资者角度判断；区分"情绪"和"事实"；对模糊利好保守评分 |
| **催化剂识别** | 公告全文或摘要 | `{is_catalyst: bool, type: "业绩预增/回购/增持/并购/重大订单/政策利好", confidence: 0-100, impact_level: "高/中/低", expiration_days: int, reasoning: "判断依据"}` | 严格标准：必须是实质利好；过滤掉"标题党"；标注置信度 |
| **投资逻辑生成** | 打分卡全量数据 | 自然语言段落（150-300字），包含：核心逻辑(为什么看好)、关键风险、建议持有周期 | 逻辑链条清晰；承认不确定性；不夸大不忽悠 |
| **风险提示生成** | 排雷数据+财务数据 | 自然语言段落（80-150字），列出最关键的3-5个风险点 | 具体、可量化、有依据 |

### 12.5 LLM 缓存与成本控制

| 策略 | 说明 |
|------|------|
| **新闻去重缓存** | 相同 URL 或标题相似度 > 0.9 的新闻，24 小时内不重复分析 |
| **分层调用** | 全市场扫描时使用简单规则（关键词匹配）做初筛；仅对 Top 50 股票调用 LLM 深度分析 |
| **批量处理** | 一次 API 调用传入该股票的全部近期新闻（最多 20 条），而非逐条分析 |
| **每日成本预估** | 按 Top 50 股票 × 5 条新闻 × 4 项分析 = 约 1000 次 API 调用 / 日 |
| **Local 降级** | 提供 local 模式：使用本地小型模型（如 Ollama + Qwen），零成本但效果略降 |

## 13. 市场环境识别模块

### 13.1 模块定位

市场环境识别模块**不直接参与股票评分**，而是：
1. 判定当前市场状态（牛市/震荡市/熊市）
2. 输出市场温度和建议仓位
3. 动态调节各因子大类的权重

### 13.2 市场状态判定

**判定标的**：上证指数、沪深300、中证500、中证1000（四个维度综合判断）。

| 判定指标 | 权重 | 计算方法 |
|---------|------|---------|
| **均线位置** | 30% | 四个指数分别在 EMA60 和 EMA200 上方的比例。4/4 在上方 = 满分 |
| **均线方向** | 25% | EMA60 斜率方向。上斜 = 看多，下斜 = 看空 |
| **指数 60 日涨幅** | 20% | 四个指数 60 日涨幅均值 |
| **成交量趋势** | 15% | 四个指数近 20 日均量的变化方向 |
| **新高新低比** | 10% | 全市场 20 日新高股票数 vs 新低股票数 |

### 13.3 状态判定结果

| 市场温度得分 | 状态 | 特征 | 建议仓位 | 权重调节 |
|------------|------|------|---------|---------|
| ≥ 75 分 | **牛市** 🟢 | 指数多头排列，新高>新低，量能放大 | 80-100% | 技术面权重 +5%，基本面权重不变 |
| 50–74 分 | **震荡偏多** 🟡 | 指数在均线上方但方向不明 | 50-80% | 默认权重 |
| 25–49 分 | **震荡偏空** 🟠 | 指数在均线下方，新低增多 | 30-50% | 基本面权重 +5%，技术面权重 -5% |
| < 25 分 | **熊市** 🔴 | 指数空头排列，量缩，恐慌情绪 | 0-30% | 基本面权重 +10%，技术面权重置零，严格风控 |

### 13.4 权重自适应机制

```
牛市（≥75分）:
  基本面 35% + 技术面 30% + 资金面 10% + 行业 10% + 消息面 10% + 催化剂 5% + 市场环境 5%
  → 技术面 +5%，从基本面挪出（趋势惯性下技术信号更可靠）

震荡市（25-74分）:
  基本面 35% + 技术面 25% + 资金面 10% + 行业 10% + 消息面 10% + 催化剂 5% + 市场环境 5%
  → 默认权重

熊市（<25分）:
  基本面 45% + 技术面 0% + 资金面 15% + 行业 10% + 消息面 15% + 催化剂 10% + 市场环境 5% = 100%
  → 技术面25%分配给：基本面+10%、资金面+5%、消息面+5%、催化剂+5%
  → 基本面提权防错杀，技术面放弃（逆势技术信号无效）
  → 建议仓位 ≤ 30%（通过仓位控制风险，而非评分公式）
```

### 14.1 融合架构

```
  │ 35%      │ │ 25%      │ │ 10%      │ │ 10%      │ │ 10%      │
  └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘
        │             │           │           │             │
        └─────────────┴───────────┴───────────┴─────────────┘

                          │  Final Score │
                          │  0-100       │
```

### 14.2 默认权重（震荡市基准）

| 维度 | 权重 | 说明 |
|------|------|------|
| 基本面 (Fundamental) | 35% | 估值+成长+质量+预期 |
| 技术面 (Technical) | 25% | 趋势+动量+量价+形态+多周期+共振加成 |
| 资金行为 (Capital Flow) | 10% | 北向+主力+筹码 |
| 行业景气 (Industry) | 10% | 行业强度+资金+估值变化+热度 |
| 消息面 (News) | 10% | 资讯情绪+社区情绪 |
| 催化剂 (Catalyst) | 5% | 事件驱动（无催化剂时 50 分基数） |
| 市场环境 (Market Regime) | 5% | 牛市高仓位→高分，熊市低仓位→低分 |
| **合计** | **100%** | 牛市/熊市时各维度权重额外 ±5% 动态调节 |

### 14.3 缺失维度处理

当某只股票因数据缺失无法计算某个维度时：

| 缺失维度 | 处理方式 |
|---------|---------|
| 基本面缺失（新股、数据未出） | 该维度权重分配给其他已计算维度，按比例缩放 |
| 技术面缺失（上市不足 60 天） | 技术面权重分配给基本面（新股看基本面） |
| 资金面缺失（无北向标的） | 北上权重归零，主力+筹码按比例放大 |
| 消息面缺失（近 7 日无新闻） | 该维度权重分配给基本面和技术面 |
| 催化剂缺失 | 默认赋值 50 分，不额外分配权重 |

### 14.4 最终得分公式

```
Final Score = 
    Fundamental × W_fund
  + Technical  × W_tech
  + Capital    × W_cap
  + Industry   × W_ind
  + News       × W_news
  + Catalyst   × W_cat
  + Regime     × W_regime

其中 W_fund+W_tech+W_cap+W_ind+W_news+W_cat+W_regime = 100%
默认: 35+25+10+10+10+5+5 = 100%
牛市/熊市时各维度权重额外 ±5% 动态调节（见 13.4）

Top 1-2 按 Final Score 降序输出
```

### 14.5 置信度与推荐等级

| Final Score | 置信度 | 推荐等级 | 含义 |
|------------|--------|---------|------|
| ≥ 85 分 | ⭐⭐⭐⭐⭐ | **S** | 极高质量：多维共振，建议重点关注 |
| 75–84 分 | ⭐⭐⭐⭐ | **A** | 高质量：多数维度优秀，少数维度中性 |
| 60–74 分 | ⭐⭐⭐ | **B** | 中等质量：部分维度优异但存在短板 |
| < 60 分 | — | 不推荐 | 各维度均无突出表现，今日无标的 |

**如果今日最高分 < 60 分，系统输出"今日无符合条件的标的"，保持宁缺毋滥原则。**

### 14.6 推荐周期判断

| 主导维度 | 推荐周期 | 逻辑 |
|---------|---------|------|
| 基本面+催化剂主导 | 中线（1-3 月） | 基本面改善需要时间兑现 |
| 技术面+资金面主导 | 短线（1-4 周） | 技术信号+资金共振 |
| 基本面+行业景气主导 | 长线（3-12 月） | 基本面优秀+行业向上趋势 |
| 催化剂主导 | 短线（事件窗口） | 催化剂有有效期 |

## 15. 卖出逻辑体系

### 15.1 模块定位

卖出逻辑是选股系统的**完整闭环**。选股只是开始，知道什么时候卖同样重要。该模块在回测中用于模拟卖出，在实盘中以建议形式呈现（V1 阶段仅回测+建议，不做实际下单）。

### 15.2 卖出触发条件

#### 15.2.1 时间止盈

| 周期 | 条件 | 逻辑 |
|------|------|------|
| 20 日 | 持仓 20 个交易日，收益率 ≥ 5% | 短线达到目标，获利了结 |
| 60 日 | 持仓 60 个交易日，收益率 ≥ 5% | 中线持有到期，无论盈亏都卖出 |
| 120 日 | 持仓 120 个交易日 | 长线最终期限，强制平仓 |

**处理方式**：哪个先触发就执行哪个。20 日和 60 日需要满足收益率条件，120 日是硬时限。

#### 15.2.2 技术止损

| 类型 | 条件 | 逻辑 |
|------|------|------|
| EMA200 跌破 | 收盘价 < EMA200（日线）| 跌破长期均线 = 趋势反转信号 |
| 趋势破坏 | EMA20 下穿 EMA50（日线死叉）+ 价格在两条均线下方 | 短期趋势走坏 |
| MACD 顶背离 | 股价新高 + MACD 未新高 | 上涨动能衰竭 |
| 放量下跌 | 跌幅 > 3% + 成交量 > 20日均量 2 倍 | 恐慌性抛售 |
| 移动止盈回撤 | 从持仓最高点回撤 > 8% | 保护利润 |

#### 15.2.3 基本面止损

| 类型 | 条件 | 触发周期 |
|------|------|---------|
| 业绩恶化 | 最新财报净利润同比下滑 > 30% | 财报公告次日 |
| 财务风险 | 新增证监会立案调查、审计意见为非标、商誉减值 | 事件发生当日 |
| 预期下修 | 业绩预告方向转为预减/首亏 | 预告公告次日 |

#### 15.2.4 盈利止盈（分级减仓）

| 收益率 | 卖出比例 | 逻辑 |
|--------|---------|------|
| ≥ 10% | 卖出 30% | 锁定部分利润 |
| ≥ 20% | 再卖出 30%（累计 60%） | 继续减仓 |
| ≥ 30% | 全部卖出 | 完全止盈 |

**叠加规则**：分级止盈与移动止盈同时生效，哪个条件先触发就执行哪个。

### 15.3 卖出优先级

```
1. 基本面止损（最高优先级，不可忽略）
2. 技术止损（趋势反转信号）
3. 移动止盈回撤（保护利润）
4. 分级盈利止盈（主动获利）
5. 时间止盈（到期了结）
```

### 15.4 回测中的卖出模拟

在回测中：
- 每日检查所有持仓的卖出条件
- 条件触发 → 以次日开盘价（考虑滑点）卖出
- 同一天触发多个条件 → 取最高优先级的条件作为卖出原因记录
- 卖出后资金释放，可在下个交易日参与新的买入

## 16. 因子评价体系

### 16.1 模块定位

因子评价体系用于**持续监测各因子的预测能力**，自动识别失效因子并调节权重。这是系统的"自我进化"机制。

### 16.2 IC 分析

| 指标 | 计算方法 | 用途 |
|------|---------|------|
| **IC（Information Coefficient）** | 因子得分与未来 N 日收益率的 Spearman 秩相关系数 | 衡量因子预测方向是否正确 |
| **ICIR（IC Information Ratio）** | IC 均值 / IC 标准差 | 衡量因子预测能力的稳定性 |
| **正 IC 率** | IC > 0 的月份数 / 总月份数 | 因子方向的稳定性 |

**IC 评估周期**：
- 日频计算 IC（每日横截面）
- 月频汇总 IC 均值和 ICIR
- 滚动 12 个月窗口评估

### 16.3 分层回测

| 步骤 | 方法 |
|------|------|
| 1. 分组 | 按因子得分将股票分为 5 组（Q1 最低分 → Q5 最高分） |
| 2. 组合构建 | 每组等权构建组合，月频调仓 |
| 3. 收益计算 | 计算每组未来 20 日的平均收益率 |
| 4. 多空收益 | Q5（高分） - Q1（低分）的多空收益 |
| 5. 单调性检验 | Q1→Q5 收益是否单调递增（越高分的组收益越高） |

### 16.4 因子贡献率

使用 **加权 IC 贡献法**：

```
因子贡献率 = (因子权重 × |因子IC|) / Σ(所有因子权重 × |因子IC|)

因子贡献率 > 30% → 核心因子，保护权重
因子贡献率 < 5%  → 边缘因子，考虑降权或淘汰
```

### 16.5 自动降权与淘汰

| 条件 | 动作 | 频率 |
|------|------|------|
| 滚动 12 个月 ICIR < 0.1 | 因子权重降低 50% | 每月检查 |
| 滚动 12 个月正 IC 率 < 55% | 因子标记为"方向不稳定" | 每月检查 |
| 滚动 24 个月 IC 均值 ≈ 0（不显著不为0） | 因子建议淘汰，人工确认 | 每季度检查 |
| 因子相关系数 > 0.8（与其他因子高度共线性） | 保留 IC 更高的那个，另一个降权 | 每季度检查 |

**重要**：自动降权执行，淘汰需人工确认（config 中标记 `deprecated: true` 后不再计算）。

### 16.6 因子评价报告示例


## 17. 风险控制系统

### 17.1 硬性过滤（选股前执行）

| 过滤项 | 条件 | 优先级 |
|--------|------|--------|
| ST / *ST | 自动淘汰 | P0 |
| 退市整理期 | 自动淘汰 | P0 |
| 上市不满 180 天 | 自动淘汰（次新股波动大、数据少） | P0 |
| 停牌 | 停牌期间排除，复牌后观察 3 天 | P0 |
| 收到证监会立案调查 | 自动淘汰 | P0 |
| 近 1 个月有重大负面公告（财务造假、实控人出事等） | 自动淘汰 | P0 |
| 黑名单 | 用户维护的黑名单，自动淘汰 | P0 |

### 17.2 流动性过滤

| 条件 | 阈值（可配置） | 说明 |
|------|--------------|------|
| 日成交额 | ≥ 3000 万元 | 剔除日均成交额过低的股票 |
| 换手率 | ≥ 0.3% | 剔除完全无人交易的股票 |
| 流通市值 | ≥ 20 亿元 | 剔除微盘股（流动性风险大） |

### 17.3 行业集中度限制

| 条件 | 阈值 |
|------|------|
| 单一行业占比（在 Top N 组合中） | ≤ 30% |
| 如果某行业在 Top 5 中已有 2 只以上 | 同行业排名第 3 及之后的不入选 |

### 17.4 异常公告过滤

| 异常类型 | 处理 |
|---------|------|
| 审计意见为"保留意见"/"无法表示意见" | 淘汰 |
| 近 1 个月有业绩变脸（预增→预减/亏损） | 淘汰 |

## 19. 回测验证体系

### 19.1 回测模式

| 模式 | 说明 |
|------|------|
| **逐日回测** | 从起始日期到终止日期，每日运行选股逻辑 → 模拟买入 → 跟踪卖出 → 记录收益 |
| **固定持仓期** | 每日买入 Top N，持有固定天数后卖出。分别测试 20日/60日/120日 持仓期 |
| **卖出规则回测** | 使用完整的卖出逻辑体系（第15章），测试止盈止损规则的有效性 |

### 19.2 回测假设

| 假设项 | 设定 |
|--------|------|
| 成交价格 | T+1 开盘价 |
| 滑点 | 0.1%（可配置） |
| 佣金 | 万分之 2.5（可配置） |
| 印花税 | 卖出 0.1%（A股实际税率） |
| 涨跌停限制 | 涨停买不到（跳过），跌停卖不掉（下一日继续挂单） |
| 最大持仓数 | 2 只（可配置） |
| 初始资金 | 100 万元（可配置） |
| 单只仓位 | 等权分配 |

### 19.3 输出指标

| 指标类别 | 具体指标 |
|---------|---------|
| **收益指标** | 年化收益率、累计收益率、超额收益（vs 沪深300 / 中证500） |
| **风险指标** | 最大回撤、年化波动率、夏普比率、Calmar 比率、最大回撤持续天数 |
| **交易指标** | 胜率、盈亏比、平均持仓天数、年化换手率、总交易次数 |
| **归因指标** | 各因子 IC 时序、行业暴露分析、市值暴露分析、月度收益分布 |
| **稳定性指标** | 滚动 12 个月收益、上半年 vs 下半年收益、牛熊市分段收益 |

### 19.4 参数优化

| 优化项 | 方法 | 约束 |
|--------|------|------|
| 因子权重 | 网格搜索，步长 5%，总和 = 100% | 单因子权重 ∈ [0%, 40%] |
| 阈值参数 | 网格搜索或贝叶斯优化 | 在合理区间内搜索 |
| 持仓数量 | 测试 1/2/3/5 只的表现 | 不超过 5 只 |
| 卖出参数 | 测试不同的止盈止损比例 | 保持逻辑一致性 |

**注意**：优化在样本内（如 2020-2023）进行，在样本外（2024-2025）验证，防止过拟合。

## 22. 边界与不做的事

### V2.0 范围外（明确不做）

| 项目 | 原因 |
|------|------|
| Web 界面 / 仪表盘 | 坚持 CLI 定位 |
| 实时推送通知 | V2 阶段预留接口，暂不实现 |
| 自动下单 / 程序化交易 | 系统仅做选股建议，不连接券商 API |
| 深度学习因子挖掘 | 因子以可解释的经典量化因子为主 |
| 多市场支持 | 仅 A 股 |
| 数据库持久化（MySQL/PostgreSQL） | 本地 Parquet 文件缓存足够 |
| 实时行情（Level-2 / Tick 级） | 每日盘后日线数据即可 |
| 高频交易 / 日内策略 | 选股周期为短线(数周)到长线(数月) |
| 移动端 App | 无计划 |

### V3.0 可考虑（未来方向）

- Web Dashboard（轻量级，展示选股历史+回溯统计）
- 飞书/微信推送通知
- 对接券商 API 实现半自动化下单（需人工确认）
- 更丰富的多因子模型（Barra 风格）
- 实时盘中监控与预警
